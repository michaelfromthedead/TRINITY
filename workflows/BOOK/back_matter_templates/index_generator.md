# Index Generator — Algorithm Document

**Used by:** FORMATTER (§5.2)
**Output:** `back/index.md`
**Source spec:** BOOK_PRODUCTION.json §roles.FORMATTER.responsibilities.back_matter_generation.index_note

---

## Purpose

Extract key terms from chapter files and produce a preliminary index with chapter:section locators. The result is a starting point for human refinement — not a publication-ready index.

---

## Algorithm

### Step 1 — Collect candidate terms from STORYBOARD.md (highest priority)

If `STORYBOARD.md` is available in the project root:

1. Read all `#### Concepts Introduced` sections.
2. Extract all concept names in `[ALL_CAPS_WITH_UNDERSCORES]` format.
3. Convert to human-readable form: `NON_LOCALITY_PROBLEM` → `non-locality problem`.
4. These are first-priority index entries — they will appear in the index regardless of frequency.

Record the concept-to-chapter mapping (concept was introduced in CH_NN).

---

### Step 2 — Collect typographically emphasized terms

Scan all chapter files for:

**Bold terms:**
```
Regex: \*\*([^*]{2,50})\*\*
Extract: the emboldened text
Filter: exclude bold used for section headings (these start at line start after markdown heading markers)
```

**Italic terms (typically technical or introduced terms):**
```
Regex: \*([^*]{2,30})\*  OR  _([^_]{2,30})_
Extract: the italicized text
Filter: exclude italic used for book titles (typically longer, contain spaces common to titles)
```

---

### Step 3 — Collect explicitly defined terms

Scan chapter files for definition-marker patterns:

```
Pattern 1: "we (?:define|call) ([A-Za-z][^.]{2,40}) as "
Pattern 2: "([A-Za-z][^.]{2,40}) (?:is defined as|refers to|is the term for) "
Pattern 3: "([A-Za-z][^,.]{2,30}), or [A-Za-z][^,.]{2,30}, (?:is|are) "
Pattern 4: Named em-dash definition: "([A-Za-z][^—]{2,40}) — [A-Za-z]"
```

Extract the term (group 1) from each match.

---

### Step 4 — Collect high-frequency terms

For terms not already collected via Steps 1-3:

1. Tokenize each chapter file into words/phrases.
2. Remove stop words: a, an, the, and, or, but, in, on, at, to, for, of, with, by, from, is, are, was, were, be, been, being, it, its, this, that, these, those, etc.
3. Count occurrences of each remaining term across all chapter files.
4. Threshold: include terms appearing in ≥ 3 different chapter files OR appearing ≥ 8 times total in a single chapter.
5. For multi-word phrases, check for noun phrases up to 3 words that co-occur consistently (bigrams and trigrams with frequency ≥ 5).

---

### Step 5 — Collect proper nouns

Scan for:

- Physicist/mathematician names: sequences of capitalized words in running text (not at sentence start) that match the pattern of names (e.g., "Larmor", "Noether", "Bell", "Einstein", "Dirac").
- Theorem and experiment names: "X's theorem", "the X experiment", "the X paradox".
- Named physical quantities: "Larmor frequency", "EPR correlation", "Bell inequality".

---

### Step 6 — Build locator map

For each collected term (from all steps above):

1. Identify every chapter file where the term appears.
2. Within each chapter file, identify the section(s) (H2 headings) where the term appears.
3. Record as: `term → [CH_NN §section_number, CH_MM §section_number, ...]`

For STORYBOARD-derived concepts, the locator for the concept's origin chapter is given priority (mark it as the primary reference, e.g., with bold in the index entry).

---

### Step 7 — Normalize and deduplicate

- Normalize case: use the term's most common capitalization pattern.
- Merge variants: "angular momentum" and "Angular Momentum" → one entry.
- Merge related forms: "spin" and "spins" → "spin" (lemmatize to root form where unambiguous).
- Flag ambiguous merges: if "Bell" could refer to multiple physicists, note it.

---

### Step 8 — Sort and produce

Sort alphabetically by first word of term (A→Z). Ignore leading articles (a, an, the) for sort order.

Produce `back/index.md`:

```markdown
# Index

angular momentum, CH_03 §3.1, CH_05 §5.2
Bell inequality, CH_01 §1.2, CH_02 §2.3
EPR thought experiment, **CH_01 §1.1**, CH_03 §3.3
Larmor frequency, CH_02 §2.2, CH_02 §2.3
non-locality problem, **CH_01 §1.1**, CH_03 §3.3
Noether's theorem, CH_02 §2.3
spin precession, CH_02 §2.1, CH_02 §2.2
...

---

*NOTE: This is a preliminary index generated by FORMATTER from automated key-term extraction.
Bold locators indicate the chapter where the concept is first introduced or formally defined.
Human review and refinement is required before publication.*
```

---

## Ambiguous cases — flagging protocol

When a term is ambiguous (same word used in multiple senses, or unclear whether it should be indexed), add a flag inline:

```
spin [AMBIGUOUS: used both as "particle spin" and colloquially in CH_01 §1.3 — human review needed], CH_01 §1.1, CH_02 §2.1, CH_03 §3.1
```

---

## Known limitations

- This algorithm produces a working draft. Professional index quality requires human judgment for:
  - Appropriate level of granularity (what deserves an entry vs. what doesn't)
  - Subentries and cross-references ("see also" entries)
  - Variant forms and synonyms the algorithm may not catch
  - Terms that should NOT be indexed despite high frequency (transition words used technically)
- Mathematical symbols (∇, ℏ, σ, etc.) require manual index entries — this algorithm handles text terms only.
- Locators are at section level, not page level — actual page locators are filled by the typesetting pipeline after pagination.
