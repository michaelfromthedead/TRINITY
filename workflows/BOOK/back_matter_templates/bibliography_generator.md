# Bibliography Generator — Algorithm Document

**Used by:** FORMATTER (§5.1)
**Output:** `back/bibliography.md`
**Source spec:** BOOK_PRODUCTION.json §roles.FORMATTER.responsibilities.back_matter_generation.bibliography_note

---

## Purpose

Collect all citations from chapter files, deduplicate, format per STYLE template convention, sort alphabetically, and produce a formatted bibliography.

---

## Algorithm

### Step 1 — Scan chapter files for citation patterns

Read each file in `chapters/` sequentially. For each file, scan line by line for the following patterns. When a pattern matches, extract the citation information and record the source file + approximate location.

**Pattern A — Parenthetical author-year:**
```
Regex: \(([A-Z][A-Za-z\-]+(?:\s+et\s+al\.)?),\s+(\d{4}[a-z]?)\)
Examples: (Smith, 2019), (Dirac, 1928), (Bell et al., 1964)
Extract: author_string, year
```

**Pattern B — Inline author-year:**
```
Regex: [A-Z][A-Za-z\-]+(?:\s+et\s+al\.)?\s+\((\d{4}[a-z]?)\)
Examples: Smith (2019) showed..., Dirac (1928) demonstrated...
Extract: author_string (from preceding word(s)), year
```

**Pattern C — Numbered footnote reference:**
```
Regex: \[\^(\d+)\](?!:)
Match: [^1], [^2], etc. (reference markers, not definition lines)
Then find the corresponding definition:
Definition regex: \[\^(\d+)\]:\s+(.+)
Extract: footnote_number, full_citation_text
```

**Pattern D — Explicit reference list entries:**
If the chapter file ends with a section `## References` or `## Notes`, collect all items in that section as citation entries.

**Pattern E — Ambiguous or unknown:**
If a citation-like structure is found but does not match A-D clearly, record it as AMBIGUOUS and note it in the generator's findings section.

---

### Step 2 — Deduplicate

Normalize citation keys:
- For author-year citations: key = `lowercase(first_author_surname)_year`
- For footnote citations: key = hash of the full citation text
- Merge duplicates: if the same work is cited multiple times (same key), keep one entry and record the count.

---

### Step 3 — Resolve and format

For author-year pattern matches (Patterns A, B), FORMATTER has only the author and year — not the full bibliographic information. FORMATTER must:

1. Check if the full citation appears elsewhere in the chapter file (e.g., in a References section).
2. If found: use the full citation text.
3. If not found: produce a placeholder entry:
   ```
   [CITATION_UNRESOLVED: Author (Year) — full bibliographic details not found in chapter files. Author must supply.]
   ```
   Flag this as a REQUIRED_AUTHOR_ACTION in the bibliography file.

For footnote citations (Pattern C), the full citation text is available in the footnote definition. Use it directly.

---

### Step 4 — Format each citation

Apply the STYLE template's citation convention if declared in BOOK_MANIFEST.json.

If STYLE_ACADEMIC_EXPLORATORY or similar: inline parenthetical style with full reference list using APA-like format:
```
Author, A. A., & Author, B. B. (Year). *Title of work*. Publisher.
Author, A. A. (Year). Title of article. *Journal Name*, volume(issue), pages.
```

If STYLE_ACADEMIC_METASTUDY or similar: numbered reference list is more common; consult the STYLE template's citation convention section.

If no STYLE template declares a citation convention, use this generic academic default:
```
Author Surname, First Initial. (Year). *Title*. Publisher/Journal, volume(issue), pp. pages.
```

---

### Step 5 — Sort and produce

Sort all resolved citations alphabetically by first author's surname (A→Z).

For works with the same first author, sort by year (ascending).

Produce `back/bibliography.md`:

```markdown
# Bibliography

Author, A. A. (Year). *Title of work*. Publisher.

Author, B. B., & Author, C. C. (Year). *Title of article*. *Journal Name*, volume(issue), pages.

...

[CITATION_UNRESOLVED: ...] (if any)
```

---

### Step 6 — No citations case

If zero citation patterns were found across all chapter files:

```markdown
# Bibliography

NOTE: No citations were detected in the chapter files using FORMATTER's citation pattern recognition.
This may indicate:
  (a) The manuscript uses a citation style not covered by FORMATTER's patterns (Pattern A-D).
  (b) The manuscript does not use citations.
  (c) Citations are present but use non-standard formatting.

If the manuscript uses citations, the author must supply a bibliography manually.
If no citations are used, remove 'bibliography' from BOOK_MANIFEST.json → structure.back_matter.
```

Flag in FORMATTER report as `NO_CITATIONS_FOUND`.

---

## Output format

```markdown
# Bibliography

[Alphabetically sorted formatted entries]

---

*Generated by FORMATTER from automated citation pattern extraction. Human review expected before publication.*
```

---

## Known limitations

- Does not handle multi-volume works, conference proceedings with editors, or translated works without STYLE template guidance.
- Does not access external databases to resolve author-year citations to full bibliographic records.
- Non-Latin scripts in author names may not sort correctly — flag these for human review.
- Citation patterns within code listings or LaTeX math environments may produce false positives — these are filtered by checking that the surrounding context is prose, not code.
